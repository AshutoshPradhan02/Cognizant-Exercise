Bootstrap 5 Setup Exercises

Exercises solved: 1.1, 1.2, 2.1, 2.2.

This page includes CDN setup, local setup, Bootstrap folder structure, and the JS bundle concept.

Exercise 1.1: Bootstrap CDN Setup

Create a basic HTML page and link Bootstrap 5 via CDN.

Why use CDN? CDN delivery is fast, globally cached, and ideal for prototypes and interviews.

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.7/dist/css/bootstrap.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.7/dist/js/bootstrap.bundle.min.js"></script>

The CDN example here demonstrates Bootstrap CSS and JS loading without local files.

Exercise 1.2: Local Bootstrap Setup

Set up a project using downloaded Bootstrap files and reference them from local folders.

Local setup command example:

npm init -y
npm install bootstrap@5.3.7 bootstrap-icons sass --save-dev

Local HTML references:

<link href="bootstrap/css/bootstrap.min.css" rel="stylesheet">
<script src="bootstrap/js/bootstrap.bundle.min.js"></script>

Exercise 2.1 and 2.2: Bootstrap structure and bundle

Explore the purpose of Bootstrap CSS, JS, and icon files. Use bootstrap.bundle.min.js when you want Popper included with Bootstrap plugins.

  • CSS includes the responsive grid, components, typography, and utility classes.
  • JS provides interactive features like collapse, modal, dropdown, and toast.
  • Icons are optional vector icons from the Bootstrap Icons package.
Bootstrap Bundle includes Popper.js, so you do not need a separate Popper import for dropdowns, tooltips, or popovers.
