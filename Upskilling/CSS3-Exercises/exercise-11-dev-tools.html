Exercise 11: Debug and Test with Dev Tools and VS Code

Scenario

Some users report layout breaking on smaller screens. Your design looks perfect on your desktop, but it doesn't work correctly on mobile devices. You need to use developer tools to identify and fix these issues.

Objective

Learn to use Chrome DevTools and VS Code to debug CSS issues, test responsive design, and troubleshoot styling problems like a professional developer.

Chrome DevTools - Inspector Tab

Step 1: Open DevTools and Select Inspector

  1. Windows/Linux: Press F12 or Ctrl+Shift+I
  2. Mac: Press Cmd+Option+I
  3. The DevTools panel opens at the bottom or right side
  4. Click the "Inspector" tab (first tab, looks like a cursor)

Step 2: Select an Element

  1. Click the "Select Element" icon (top-left, looks like a cursor in a box)
  2. Hover over any element on the page - it highlights and shows dimensions
  3. Click to select it - the HTML and CSS appear in the inspector
  4. The Box Model diagram shows margin, border, padding, and content

Step 3: View CSS Rules

  1. In the "Styles" tab, see all CSS rules applied to the element
  2. Rules are listed in order: most specific rules first, then less specific
  3. Strikethrough rules mean they're overridden by more specific rules
  4. Hover over property values to see them highlighted on the page

Example: Debugging a Button

Problem: Button has conflicting styles
In DevTools, you might see:
✗ padding: 0.5rem; /* strikethrough - overridden */
✓ padding: 1rem; /* active rule - this one applies */
Solution: Find why the first rule is being overridden. Check selector specificity!

Chrome DevTools - Edit Styles Live

Live Edit CSS Properties

  1. In the Styles panel, double-click any CSS value
  2. Type a new value and press Enter
  3. The change appears immediately on the page
  4. These changes are temporary - they refresh away when you reload
  5. Use this to test fixes before modifying your source code

Example: Testing Responsive Changes

Original: width: 100%;
Change to: width: 90%;
See the effect immediately!

Original: display: flex;
Change to: display: block;
Notice how the layout changes in real-time!

Chrome DevTools - Device Toolbar

Test Responsive Design

  1. Open DevTools (F12)
  2. Click the Device Toolbar icon (top-left, looks like a phone/tablet)
  3. Choose a device from the dropdown or set custom dimensions
  4. Your page now shows as it would on that device
  5. Resize the window by dragging the edge to test at different widths

Device options:

iPhone 12 iPad Galaxy S10 Custom (320×568)

Chrome DevTools - Network Tab

Check if CSS Files Load

  1. Open DevTools and go to the "Network" tab
  2. Reload the page (Ctrl+R or Cmd+R)
  3. You see all files the page loads (HTML, CSS, JS, images, fonts)
  4. Look for .css files - check if they load successfully
  5. Red/orange indicates errors (404 Not Found, etc.)
  6. Click a CSS file to see details like file size and load time

Common Issues

  • 404 errors: CSS file path is wrong
  • CORS errors: External CSS is blocked by security
  • File doesn't appear: Link tag is missing from HTML
  • Slow loading: CSS file is too large or server is slow

VS Code - CSS Debugging & Linting

Syntax Errors in VS Code

  1. VS Code underlines CSS errors in red squiggly lines
  2. Hover over the error to see what's wrong
  3. Common errors: Missing semicolon, wrong property name, invalid value
  4. Install "CSS Peek" extension to jump to CSS from HTML

Common CSS Debugging Scenarios

Scenario 1: Styles Not Appearing

Problem: You changed CSS but nothing happened

Check:
1. Did you save the file?
2. Is the CSS file linked correctly? Check path in <link> tag
3. Did you reload the page? (Ctrl+R)
4. Is your selector specific enough? Check DevTools
5. Is a more specific rule overriding your styles?
6. Try adding !important temporarily (for debugging only!)

Scenario 2: Layout Breaking on Mobile

Problem: Site looks good on desktop but breaks on mobile

Check:
1. Open DevTools Device Toolbar
2. Select iPhone size and check where it breaks
3. Are images too large? Set max-width: 100%
4. Is text too small? Min 16px on mobile
5. Are margins/padding too large for small screens?
6. Do you have media queries for mobile?

Scenario 3: Unwanted Spacing or Alignment

Problem: Elements have extra margin or padding, or alignment is off

Check:
1. Select the element in DevTools
2. Look at the Box Model diagram
3. Identify which layer is causing the issue (margin/padding/border)
4. Edit the value in DevTools to test fix
5. Apply the fix to your CSS file

Debugging Workflow

Step Action What to Look For
1. Identify Issue Look at problem element on page Visual anomaly (wrong color, size, position)
2. Open DevTools Press F12 DevTools panel appears
3. Select Element Click selector icon, click element HTML is highlighted, styles appear
4. Inspect Styles Look at Styles tab Which rules apply? Are any strikethrough?
5. Check Box Model Look at Box Model diagram Is margin/padding/border causing issue?
6. Test Fix Edit CSS value in DevTools Does the change fix the problem?
7. Update Code Modify CSS file in editor Save and reload page

Best Practices for Debugging

  • Always reload: F5 or Ctrl+R after making changes
  • Test on multiple devices: Use DevTools Device Toolbar
  • Check file paths: Use Network tab if files don't load
  • Test live in DevTools: Before changing source code
  • Use browser cache carefully: Hard refresh (Ctrl+Shift+R) to clear cache
  • Document your fixes: Add comments explaining why you made changes
  • Check specificity: Lower specificity often solves override issues
