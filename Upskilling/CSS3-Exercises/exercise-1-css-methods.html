Exercise 1: Why CSS? Inline vs. Internal vs. External

Scenario

The designer wants you to experiment with different ways to apply styles to understand the pros and cons of each method.

Objective

Understand the three CSS inclusion methods and their impact on code organization and maintainability.

Task Breakdown

1. Apply Inline Style

Make one heading red using an inline style attribute.

This heading uses an inline style (color: red)

<h5 style="color: red;">This heading uses an inline style</h5>

2. Use Embedded <style> Tag (Internal CSS)

Define body background color in the <head> section using <style> tag.

<style>
  body {
    background-color: #f9f9f9;
  }
</style>

Current page: Notice the light gray background color applied to this page.

3. Link External Stylesheet

Link an external stylesheet and move reusable styles there.

<link rel="stylesheet" href="css/styles.css">

Current page: All reusable styles from css/styles.css are applied to this page, including header, navigation, and all styling classes.

Comparison Table

Method Syntax Pros Cons Use Case
Inline <element style="..."> Quick testing, immediate effect Not reusable, hard to maintain, clutters HTML Quick fixes, testing only
Internal (Embedded) <style>...</style> Centralized for single page, cleaner than inline Not reusable across pages, increases HTML file size Single-page projects
External <link href="styles.css"> Highly reusable, cacheable, best for SEO, clean HTML Requires separate file, one more HTTP request Professional projects, multi-page sites

Best Practice Comment Structure

When using external stylesheets, organize your CSS with comments:

/* ---- Header Styles ---- */
header {
  background-color: #667eea;
  color: white;
}

/* ---- Navigation Styles ---- */
nav {
  background-color: #333;
}

Key Takeaways

  • Use inline styles only for quick testing or single-use exceptions
  • Use internal styles for single-page applications or templates
  • Use external stylesheets for professional, maintainable projects
  • Always comment your CSS to label sections and improve readability
  • External CSS files are cached by browsers, improving performance

Challenge

Your Task: Create a new HTML file with both inline, internal, and external CSS applied. Use your browser's DevTools (F12) to inspect each element and see where its styles are coming from. Notice how external CSS takes precedence in the cascade.
