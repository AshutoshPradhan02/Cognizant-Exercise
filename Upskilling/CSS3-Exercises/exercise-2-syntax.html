Exercise 2: CSS Syntax and Comments

Scenario

You've joined a team and need to understand and maintain a large stylesheet. Clean, readable CSS with proper structure and comments is essential for team collaboration.

Objective

Write clean, readable CSS with proper structure, consistent indentation, and descriptive comments for easy maintenance.

CSS Syntax Basics

Basic Syntax Structure

selector {
  property: value;
  property: value;
}

Example:

h1 {
  color: #667eea;
  font-size: 2rem;
  margin-bottom: 1rem;
}

Comments in CSS

Single-line Comments

/* Style for main CTA button */

Multi-line Comments

/*
  Section: Header and Navigation
  Purpose: Style the top navigation bar
  Author: Your Name
  Last Updated: 2024
*/

Section Comments

/* ============================================
    HEADER STYLES
    ============================================ */

header {
  background-color: #667eea;
}

Formatting Guidelines

Guideline Good ✓ Bad ✗
Indentation Use 2 or 4 spaces consistently Mixing tabs and spaces
Spacing Space after property colon No space: color:#667eea;
Line Breaks One property per line Multiple properties on one line
Comments Descriptive comments above selectors Vague or no comments
Organization Group related styles together Random, scattered rules

Example of Well-Formatted CSS

/* ============================================
    BUTTON STYLES
    ============================================ */

/* Style for main CTA button */
.cta-button {
  background-color: #007BFF;
  color: white;
  padding: 0.75rem 1.5rem;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  font-size: 1rem;
  transition: background-color 0.3s ease;
}

/* Hover state for CTA button */
.cta-button:hover {
  background-color: #0056b3;
}

Real-World Example from This Project

Look at the external stylesheet css/styles.css to see how it's organized with:

  • Section headers with visual separation
  • Descriptive comments above related rule groups
  • Consistent indentation and spacing
  • Logical grouping of related styles
  • Media queries at the end for responsive design

Best Practice Checklist

  • Comment every major section of CSS
  • Use consistent indentation (2 or 4 spaces)
  • One property per line
  • Space after property colons
  • Group related styles together
  • Avoid overly specific selectors
  • Use meaningful class and ID names
  • Keep comments up-to-date with code changes

Challenge

Your Task: Open css/styles.css and study how it's organized. Then, create your own CSS file with at least 3 different styled elements. Make sure to:
• Add section headers
• Comment each major style group
• Use consistent formatting
• Apply it to an HTML page
