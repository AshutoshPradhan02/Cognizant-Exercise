Exercise 8: Box Model & Layout Control

Scenario

Event sections on the portal are cramped and need better spacing. Understanding the CSS box model is essential for controlling element spacing and layout.

Objective

Control element spacing with margin, padding, border, and outline. Understand how the box model affects layout.

The CSS Box Model

Every element has four layers in order from inside out:

  1. Content: The actual content (text, images)
  2. Padding: Space inside the border, around the content
  3. Border: Line around the padding
  4. Margin: Space outside the border, between elements

Box Model Properties

1. Margin (Outside Space)

Space OUTSIDE the border, between elements:

margin: 2rem; /* all sides */
margin: 1rem 2rem; /* vertical | horizontal */
margin: 1rem 2rem 1rem 2rem; /* top | right | bottom | left */
margin-top: 1rem;
margin-right: 2rem;
margin-bottom: 1rem;
margin-left: 2rem;
This box has 2rem margin on all sides (space between this and other elements)

2. Padding (Inside Space)

Space INSIDE the border, around the content:

padding: 1.5rem; /* all sides */
padding: 1rem 2rem; /* vertical | horizontal */
padding: 1rem 2rem 1rem 2rem; /* top | right | bottom | left */
This box has 3rem padding inside. Notice the content is pushed away from the border.

3. Border

Line around the padding:

border: 3px solid #764ba2;
border-width: 3px;
border-style: solid|dashed|dotted|double;
border-color: #764ba2;
This box has a 3px solid purple border

4. Outline

Similar to border but OUTSIDE the box model (doesn't affect layout):

outline: 3px dashed #e74c3c;
outline-offset: 5px; /* space between outline and border */
This box has a dashed outline with 5px offset

Box Sizing

box-sizing: content-box (default)

Width and height only include content. Padding and border are ADDED to the size.

width: 200px; /* content only */
padding: 20px; /* added to total width */
border: 1px solid; /* added to total width */
/* Total width = 200 + 20 + 20 + 1 + 1 = 242px */

box-sizing: border-box (recommended)

Width and height include content, padding, and border. Total size stays the same.

* {
  box-sizing: border-box; /* apply to everything */
}

width: 200px; /* includes padding + border */
padding: 20px;
border: 1px solid;
/* Total width = exactly 200px */

Box Model Summary Table

Property Position Purpose Affects Layout
Margin Outside border Space between elements Yes
Border Around padding Visual frame Yes
Padding Inside border Space around content Yes
Outline Outside border Highlight/focus indicator No

Developer Tools Inspection

In Chrome DevTools (F12):

  1. Right-click any element and select "Inspect"
  2. Look at the "Styles" tab
  3. Scroll down to see the Box Model diagram
  4. Hover over values to see them highlighted on the page
  5. Click and edit values live to test changes

Best Practices

  • Always use box-sizing: border-box for predictable sizing
  • Use margin for spacing between elements
  • Use padding for space inside elements
  • Use outline for focus indicators (accessibility)
  • Use DevTools to debug box model issues
  • Be consistent with margin/padding conventions
  • Use visibility: hidden for toggling without reflow
  • Use display: none to completely remove elements
