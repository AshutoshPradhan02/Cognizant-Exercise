Exercise 10: Responsive Web Design with Media Queries

Scenario

Users will access the community event portal on phones, tablets, and desktops. Your site needs to be fully responsive and look great on all devices.

Objective

Create responsive layouts using media queries, flexible units, and modern CSS techniques that adapt beautifully to any screen size.

Media Query Basics

Media Query Syntax

@media (max-width: 768px) {
  /* Styles for screens 768px and smaller */
}

@media (min-width: 769px) and (max-width: 1024px) {
  /* Styles for tablets */
}

Common Breakpoints

Device Type Screen Width Media Query
Mobile (phones) 320px - 480px @media (max-width: 480px)
Tablet (small) 481px - 768px @media (max-width: 768px)
Tablet (large) 769px - 1024px @media (max-width: 1024px)
Desktop 1025px and above @media (min-width: 1025px)

Mobile-First Approach

Best Practice: Start Mobile, Add Complexity

/* Default - Mobile (320px and up) */
body {
  font-size: 14px;
}

.grid {
  display: block; /* stack on mobile */
}

/* Tablet (768px and up) */
@media (min-width: 768px) {
  body {
    font-size: 16px;
  }
  .grid {
    display: grid;
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Desktop (1024px and up) */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

Responsive Units

Flexible Sizing Units

Unit Meaning Use Case Example
% Percentage of parent Widths and flexible layouts width: 90%;
vw % of viewport width Hero sections, full-width width: 100vw;
vh % of viewport height Full-screen sections height: 100vh;
em Relative to parent font-size Typography hierarchy font-size: 1.5em;
rem Relative to root font-size Consistent spacing margin: 2rem;

Responsive Layout Examples

Hero Section (Responsive)

Welcome to Community Events

Resize the window to see the heading size adjust responsively using clamp()!

.font-responsive {
  font-size: clamp(1rem, 5vw, 2rem);
}
/* min-size | preferred-size | max-size */
/* scales smoothly between 1rem and 2rem based on 5vw */

Grid Layout (Auto-Responsive)

This grid automatically adjusts columns based on available space:

Mobile

Single column on small screens

Tablet

2-3 columns on medium screens

Desktop

Multiple columns on large screens

Flexible

Uses auto-fit for automatic responsiveness

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1.5rem;
}
/* Automatically adjusts number of columns */
/* Each column is at least 250px, but grows to fill space */

Navigation Bar (Stack on Mobile)

/* Desktop - horizontal nav */
nav ul {
  display: flex;
  gap: 2rem;
}

/* Mobile - vertical nav */
@media (max-width: 768px) {
  nav ul {
    display: flex;
    flex-direction: column; /* stack vertically */
    gap: 0;
  }
  nav ul li a {
    display: block;
    padding: 1rem;
    border-bottom: 1px solid #ddd;
  }
}

Responsive Typography

/* Desktop */
h1 { font-size: 2.5rem; }
p { font-size: 1rem; }

/* Tablet */
@media (max-width: 1024px) {
  h1 { font-size: 2rem; }
  p { font-size: 0.95rem; }
}

/* Mobile */
@media (max-width: 768px) {
  h1 { font-size: 1.5rem; }
  p { font-size: 0.9rem; }
}

Testing Responsive Design

Chrome DevTools Device Toolbar

  1. Press F12 to open DevTools
  2. Click the Device Toolbar icon (top left)
  3. Select different devices or custom dimensions
  4. Test how your layout responds to different screen sizes
  5. Check breakpoints and media queries in the Styles tab

Responsive Best Practices

  • Mobile-first: Start with mobile design, enhance for larger screens
  • Use relative units: %, em, rem instead of fixed pixels
  • Test on real devices: Simulators don't catch everything
  • Use flexbox and grid: For flexible layouts
  • Optimize images: Use srcset for different resolutions
  • Touch-friendly: Min 44px height for touch targets
  • Readable text: Min 16px font size on mobile
  • Viewport meta tag: Always include in <head>

Essential Viewport Meta Tag

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This MUST be in your <head> for responsive design to work properly!

Challenge

Your Task: Build a fully responsive event portal page with:
• Mobile-first design that stacks on small screens
• 2-3 breakpoints (mobile, tablet, desktop)
• Responsive navigation that collapses on mobile
• Flexible typography using units like rem and clamp()
• Grid/flexbox for responsive layouts
Test in Chrome DevTools by resizing the window and using device preview!
