Exercise 6: Link and List Styling

Scenario

The default blue links and bullet lists don't match the event portal's design. You need to customize links and lists to fit the branding and improve user experience.

Objective

Customize links with pseudo-classes and create styled lists for navigation and content.

Link Pseudo-classes

Link States

Links have four pseudo-classes that represent different states:

a:link { color: #667eea; } /* unvisited link */
a:visited { color: #764ba2; } /* visited link */
a:hover { color: #764ba2; } /* mouse over link */
a:active { color: #555; } /* link being clicked */

Styled Link Example

Here's a link with all pseudo-classes styled: Hover over me!

The link changes color on hover and shows an underline animation.

Focus and Focus-Visible (Accessibility)

Important for keyboard navigation:

a:focus {
  outline: 2px solid #667eea;
  outline-offset: 2px;
}

a:focus-visible {
  box-shadow: 0 0 0 3px rgba(102, 126, 234, 0.3);
}

List Styling

List Style Type

Change the marker used for lists:

list-style-type: disc; /* solid circle (default) */
list-style-type: circle; /* hollow circle */
list-style-type: square; /* filled square */
list-style-type: decimal; /* numbers 1, 2, 3... */
list-style-type: lower-alpha; /* letters a, b, c... */
list-style-type: none; /* no marker */

List Style Position

Controls whether markers are inside or outside the list item:

list-style-position: outside; /* default */
list-style-position: inside; /* marker inside text */

List Styling Summary Table

Property Values Purpose
list-style-type disc, circle, square, decimal, none, etc. Change marker style
list-style-position outside, inside Position of marker
list-style-image url('image.png') Custom image as marker
list-style Shorthand for all three Combine all properties

Link Pseudo-classes Reference

Pseudo-class When Applied Example Use
:link Unvisited link a:link { color: blue; }
:visited Visited link a:visited { color: purple; }
:hover Mouse over link a:hover { text-decoration: underline; }
:active Link being clicked a:active { color: red; }
:focus Keyboard focus a:focus { outline: 2px solid blue; }

Best Practices

  • Always include :hover and :focus states for interactive elements
  • Make visited links visually distinct so users know where they've been
  • Remove bullet points from navigation lists with list-style: none
  • Use padding and margins to space list items
  • Remove default list styles with margin: 0; padding: 0;
  • Use flex or grid to create horizontal lists
  • Ensure sufficient hit area for mobile (min 44px height)
