Exercise 3: Selectors Playground

Scenario

You need to style various elements based on IDs, classes, and element types. Understanding different selector types is crucial for writing efficient and maintainable CSS.

Objective

Master different selector types and understand when to use each one for optimal specificity and maintainability.

CSS Selector Types

1. Universal Selector (*)

Selects ALL elements on the page. Used for reset styles.

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

Use Case: Reset default browser margins and padding

2. Element Selector

Selects all instances of a specific HTML element.

h2 {
  color: #667eea;
  font-size: 1.8rem;
}

Use Case: Style all headings or paragraphs consistently

3. ID Selector (#)

Selects a single element with a specific ID. IDs must be unique.

#mainHeader {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  color: white;
}

Use Case: Style unique, important sections like the main header

Specificity: Very high (100 points)

4. Class Selector (.)

Selects all elements with a specific class. Classes are reusable.

.eventCard {
  border: 2px solid #667eea;
  border-radius: 8px;
  padding: 1.5rem;
}

Use Case: Style reusable components across multiple pages

Specificity: Medium (10 points)

5. Grouping Selector (,)

Applies the same styles to multiple selectors.

h3, p {
  color: #333;
  line-height: 1.6;
}

Use Case: Apply consistent typography to multiple element types

Advanced Selector Types

6. Descendant Selector (space)

Selects elements that are descendants of another element.

.exercise-section h3 {
  color: #764ba2;
}

Use Case: Style headings only inside specific containers

7. Child Selector (>)

Selects elements that are direct children of another element.

.exercise-section > h2 {
  border-bottom: 3px solid #667eea;
}

Use Case: Target only direct children, not nested descendants

8. Attribute Selector ([])

Selects elements based on specific attributes.

input[type="text"] {
  padding: 0.75rem;
  border: 1px solid #ddd;
}

Use Case: Style specific input types or elements with particular attributes

9. Pseudo-class Selectors (:)

Selects elements in a specific state.

a:hover {
  color: #764ba2;
}

a:visited {
  color: #ddd;
}

Common pseudo-classes: :hover, :active, :visited, :focus, :first-child, :last-child, :nth-child()

Selector Specificity Comparison

Selector Type Example Specificity Points Use Frequency
Element h2, p, div 1 Very Common
Class .eventCard 10 Very Common
ID #mainHeader 100 Rare
Inline Style style="color: red" 1000 Avoid!
!important color: red !important; Highest Emergency Only

Best Practice Guidelines

  • Use element selectors for global styling (h1, p, body)
  • Use class selectors for reusable components
  • Use ID selectors sparingly for unique page sections
  • Avoid inline styles whenever possible
  • Keep specificity low to avoid conflicts
  • Use descendant/child selectors for context-specific styling
  • Prefer classes over IDs for styling (IDs are for JavaScript)
