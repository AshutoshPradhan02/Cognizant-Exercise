## Best Practices

### CSS Organization
- Use consistent indentation (2 or 4 spaces)
- Comment your CSS sections
- Group related styles together
- Use meaningful class and ID names

### Responsive Design
- Mobile-first approach
- Use flexible units (%, em, rem, vw, vh)
- Test on multiple screen sizes
- Use DevTools Device Toolbar for testing

### Performance
- Keep CSS files organized and minimal
- Use external stylesheets for multiple pages
- Minimize use of expensive properties
- Test load times in Network tab

### Accessibility
- Ensure sufficient color contrast
- Make interactive elements keyboard accessible
- Use :focus states for keyboard navigation
- Test with screen readers

## Tips & Tricks

1. **Use Device Toolbar (F12)** to test responsive design
2. **Edit CSS live in DevTools** to test changes before saving
3. **Use box-sizing: border-box** for predictable sizing
4. **Always include viewport meta tag** for mobile responsiveness
5. **Test on real devices** not just DevTools simulators
6. **Use rem units** for consistent spacing based on root font-size
7. **Mobile-first approach** works better than desktop-first

## Troubleshooting

### Styles not appearing?
- Check if the CSS file is linked correctly
- Make sure you saved the file
- Reload the browser (F5 or Ctrl+R)
- Check DevTools for errors in the Console
- Verify selector specificity isn't being overridden

### Layout breaks on mobile?
- Open DevTools Device Toolbar (Ctrl+Shift+M)
- Check if you have media queries for small screens
- Ensure images have max-width: 100%
- Test at different breakpoints (320px, 768px, 1024px)

### CSS file not loading?
- Check Network tab in DevTools (F12)
- Verify the file path in the `<link>` tag
- Look for 404 errors in the console
- Ensure relative paths are correct
